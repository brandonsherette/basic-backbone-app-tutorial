Introduction

While building out a web application, it may seem easier to build out interactions with simple jQuery selectors and events, however as your application grows, it can get tedious to work with. One solution is to use a JavaScript framework to help separate and alleviate some of the common tasks into a more manageable model and view structure. One such framework is Backbone.

Backbone is a more or less a more simple framework that allows you quickly build out a web application using its many pre-built feature sets such as, routing, models, and views.

Tutorial Goals

  • Install essential components.
  • Build a simple web app using the Backbone.js Framework.
  • Basic understanding of the following:
    • Views
    • Models
    • Routing

Throughout this tutorial you will not only learn how to work with the Backbone.js Framework, but also show you how to use Backbone in a small and simple application. While you continue to develop this simplistic Product Manager Application, you will see how all the pieces fit together as a semi-finished product by showing you one of my app/file structures I like to use, and utilize RequireJS to modularize your application code.

This particular tutorial is separated into four different parts.

Part 1

Part 1 is the current page you are on, and will discuss installing the required files and going over the app/file structure.

Part 2

Part 2 deals with teaching some of the basics of Backbone Views and some of its various ways of working with them and their respected html templates.

Part 3

Part 3 deals with the introduction of Backbone Models and Collections while also continuing to show different aspects of views. We will have our views render with Product Models and end with dealing with looping through a collection of products and rendering them to the user.

Part 4

Finally, we end with part 4 which deals with the introduction to Backbone Routers, and flesh out the rest of the application with adding/editing/deleting products.

1.2 App/File Structure

Backbone doesn't really have a required file structure, you could in fact have all your code in one JavaScript file, however I will be going over one of the ways I like to structure my Backbone Applications, which in turn, will help give you an idea how you can structure your own apps.

Let's first open up the backbone-tutorial-walkthrough, known from now on as the root directory, and checkout what we have.

Fig 1.2 (root)

File Structure Descriptions

root/app
The folder containing all application code, such as JavaScript and vendor files/libraries.
root/styles
Where your CSS, LESS, Font, or any other styles that the application uses. Unless you want to change any styles in the application, you will not need to go into this directory since I have already added some styling to get you started.
root/index.html
Open this file in browser to open the application. There is no need to have multiple html pages, since this is a Single Page Application (SPA). In this particular tutorial, this is where you will add your html view templates at.

App Directory

Now lets open the app directory and see the structure in there.

Fig 1.3 (root/app)
root/app/collections
Stores the appliaction's Collection Classes.
root/app/models
Stores the application's Model Classes.
root/app/services
Stores the application's Service Classes.
root/app/vendor
Stores all 3rd party files, such as Backbone, Bootstrap, Underscore, and etc...
root/app/views
Stores the application's views.
root/app/app.js
Binds and configures tasks for the application, before actually starting the application.
root/app/exe.js
Waits for the document to be ready and/or device ready if the application is for a Cordova Project, before running the application (app.js).
root/app/require-config.js
Configuration file for RequireJS. Requirejs isn't required for a Backbone Application, however I like to use it for my applications. If you want to know more about RequireJS, I will eventually have an article dissucssing it in more detail, but you can learn more at http://requirejs.org/.
root/app/router.js
Configures the application router's page states.
You can take a look at any files you wish to kind of get a better understanding, some of the files I have already coded for you, such as the require-config file, so you can concentrate on just Backone related programming.
